Capabilities

Everything the default
database CLI is missing

Built for developers who spend real time in the terminal. Every feature was chosen to reduce friction, not add it.

🎨
Syntax Highlighting
Powered by Pygments. SQL keywords, strings, numbers, and comments all rendered in colour as you type.
⚡
Smart Autocompletion
Tab-completes SQL keywords alongside live table and column names fetched from your schema. Dot commands too.
🔖
Query Bookmarks
Save any query under a short name. Bookmarks persist across sessions in ~/.TerSQL_bookmarks.json.
📊
Multiple Output Formats
Switch between grid, json, csv, vertical, markdown, html, latex, and more on the fly.
🛡
Safety Guards
Safe mode blocks dangerous queries. Read-only mode prevents all writes before they reach the server.
📈
Query Profiling
Integrates with MySQL's SHOW PROFILES to show per-stage execution timing on any query.
🔒
SSL / TLS Support
Pass CA, client cert, and client key paths directly on the command line for encrypted connections.
📜
Persistent History
Up/down arrow recall across sessions. Ctrl+R for fuzzy search through the last 500 queries.
📤
Export & Tee
Export any result set to CSV or JSON. Mirror live session output to a file with .tee.
🧠
Smart SQL Correction
Type imperfect SQL — TerSQL fixes it automatically and shows the correct version.
📂
Script Execution
Run any .sql file directly with .source. Multi-statement files with comments are fully supported.
🖥
Batch Mode
Use -e to run a single query and exit — perfect for shell scripts, cron jobs, and CI pipelines.

Setup

Get running in 60 seconds

Install the dependencies and run TerSQL in any environment.

# Clone and install dependencies
$ git clone https://github.com/developersayandutta/TerSQL.git
$ cd TerSQL/v0.0.2 && pip install -r requirements.txt
 
# Start the multi-DB terminal
$ python main.py
# Make executable wrapper
$ echo 'python /path/to/TerSQL/main.py "$@"' > tersql
$ chmod +x tersql
 
# Install system-wide
$ sudo mv tersql /usr/local/bin/tersql
 
# Run from anywhere
$ tersql
$ python main.py \
  -H secure.db.example.com \
  -u myuser \
  --ssl-ca /etc/ssl/mysql/ca.pem \
  --ssl-cert /etc/ssl/mysql/client-cert.pem \
  --ssl-key /etc/ssl/mysql/client-key.pem
# Single query, exit immediately
$ python main.py -e "SELECT COUNT(*) FROM orders;"
 
# JSON output for scripts
$ python main.py --output json -e "SELECT id, name FROM products;"
 
# Export CSV to file
$ python main.py --output csv -e "SELECT * FROM customers;" > out.csv

Protection

Two layers of safety

Opt-in guards that catch mistakes before they cost you. Ideal for production databases and shared environments.

Safe Mode

Guard Rails for Dangerous Queries

Blocks DELETE and UPDATE statements without a WHERE clause entirely. Requires typing yes before executing DROP TABLE or TRUNCATE.

--safe-mode
Read-only Mode

Zero-Risk Production Access

Blocks all write operations at the terminal level — before the query reaches the server. INSERT, UPDATE, DELETE, DROP, ALTER, GRANT, and more.

--read-only
